Unveil.js

A very lightweight plugin to lazy load images for jQuery or Zepto.js

Tweet

Most of us are familiar with the Lazy Load plugin by Mika Tuupola.

This plugin is very useful and it boosts performance delaying loading of images in long web pages because images outside of the viewport (visible part of web page) won't be loaded until the user scrolls to them.

Lazy Load has some cool options such as custom effects, container, events or data attribute. If you're not gonna use any of them you can reduce the file size by leaving just the essential code to show the images.

That's what I did and this is my lightweight version of Lazy Load with support for serving high-resolution images to devices with retina displays - less than 1k.

As of version 2, a few more options have been added allowing more advanced functionality.

Usage

Use a placeholder image in the src attribute - something to be displayed while the original image loads - and include the actual image source in a "data-src" attribute.

If you want to serve high-resolution images to devices with retina displays, you just have to include the source for those images in a "data-src-retina" attribute.

You don't need to include a "data-src-retina" attribute in all the image tags, unveil is smart enough to fallback to "data-src" or even do nothing in case there isn't any "data-src" specified.


    <img src="bg.png" data-src="img1.jpg" />
    <img src="bg.png" data-src="img2.jpg" data-src-retina="img2-retina.jpg" />

If you care about users without javascript enabled, you can include the original image inside a <noscript> tag:


    <noscript><img src="img1.jpg" /></noscript>

Run the script on document ready:


    $(document).ready(function() {
      $("img").unveil();
    });

Demo

If you're on a "normal" display, unveil will load the low resolution version (800x500). In case you're on a device with a retina display, the high resolution version (1280x800 in this case) will be loaded instead.

Options

Threshold

By default, images are only loaded and "unveiled" when the user scrolls to them and they became visible on the screen.

If you want your images to load earlier than that, lets say 200px before they appear on the screen, you just have to:


    $("img").unveil({ threshold: 200 });

Debounce

While the user is scrolling or resizing the window, Unveil is continuously working on the background, recalculating the new position of each image on the page in order to find out if it has become visible in the viewport. This can become very expensive for the browser since these events are fired frequently.

We can have a huge performance boost if we wait before doing all of this computation until after a threshold of time, e.g. 100ms:


    $("img").unveil({ debounce: 100 });

Container

If the images you want to "unveil" are inside a container with a vertical scrollbar, you just have to pass your container as a jQuery object:


    $("img").unveil({ container: $("ul") });

Custom

Lets say you think Unveil doesn't do enough for you. You have a crazy (or not so crazy) use case and you need to control how your images are unveiled. If you're in this position, you can tell the plugin not to "unveil" the images automatically. That basically means you want to do the heavy lifting yourself.


    $("img").unveil({ custom: true });

This option combined with the unveil event can be extremely helpful.

Events: trigger

Unveil

You can still trigger image loading whenever you need.

All you have to do is select the images you want to "unveil" and trigger the event:


    $("img").trigger("unveil");

Lookup

It is also possible to lookup for images in the viewport that haven't been "unveiled" yet.

This can be useful, for instance, in case of a tabbed layout.


    $(window).trigger("lookup");

Events: on

Unveil

When an image becomes visible within the viewport and before it's "unveiled", an "unveil" event is fired:


    $("img")
      // register the event listener before initializing unveil
      .on("unveil", function() {
        // "this" refers to the image's DOM node
      })
      .unveil();

You can use this event to do more advanced stuff like adding some fancy entrance animations to your images or serving images by screen size with the help of the custom option.

Unveiled

When an image is "unveiled", an event with that same name is fired.

You can listen to this event if you need "to do things" after an image has been unveiled.


    $("img")
      // register the event listener before initializing unveil
      .on("unveiled", function() {
        // "this" refers to the image's DOM node
      })
      .unveil();

Events: off

You can remove all the "unveil" event handlers at any time:


    // Without custom container
    $(window).off("unveil");

    // With custom container:
    // $("img").unveil({ container: $("ul") });
    $("ul").off("unveil");

Browser support

Compatible with All Browsers and IE7+.
